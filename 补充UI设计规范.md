最终版 UI 设计规范（Notion 风格 · 细化版）

> 目标：最大限度压缩设计自由度，减少主观发挥，让不同开发/设计人员做出来的结果尽可能一致、稳定、耐看。

核心原则：不靠“画边框”，而靠「背景层级 + 间距 + 阴影/hover」来分区。




---

一、整体设计哲学（必须统一认知）

1. 这是「Notion 系」而不是「Material / Ant / Bootstrap」

几乎不使用可见边框

不靠线条分割结构

靠以下三件事制造层级：

1. 背景色差（极小）


2. 间距（padding / gap）


3. Hover / Active 状态




> ❌ 禁止：1px 灰线框

❌ 禁止：卡片描边

✅ 推荐：背景变化 + hover 灰




---

二、颜色体系（最终定稿，不可随意发挥）

1. 基础中性色（Notion / shadcn 风格）

> 仅允许使用以下色号，不得自行微调



语义	色号	使用说明

页面主背景	#FFFFFF	整个应用最底层
次级背景	#F7F7F5	hover、次级区域
轻 hover 背景	#EFEFEE	列表项 hover
强 hover / 选中	#E6E6E4	当前选中项
主文字	#2F3437	标题、正文
次级文字	#6B6F76	描述、说明
弱提示文字	#9B9FA4	placeholder
分隔语义灰	#DADADA	仅用于 icon / 禁用态，不画线


> ⚠️ 说明：

#DADADA 不是用来画边框的

页面中原则上看不到直线





---

三、边框与分隔规则（重点澄清）

1. Notion 为什么“看起来没边框”

真实情况：

99% 的区域没有 border

少数控件在特定状态下使用极弱边界（例如 focus）



2. 本项目统一规则

🚫 默认禁止

border: 1px solid ...

outline: 1px ...

卡片描边


✅ 允许的“类边框”手段

手段	用途

背景色变化	区分区域
box-shadow（极弱）	浮层、弹窗
内边距	强化模块感
hover 背景	列表分割


唯一允许的 box-shadow

0 1px 2px rgba(0,0,0,0.04)

仅用于：

弹窗

下拉菜单

浮层播放器（展开态）



---

四、页面结构总览

页面组成

三个标签页 + 一个账号页

1. 搜索页（主页面）


2. 收藏页


3. 播放历史页


4. 登录 / 账号页



全局组件：

顶部全局搜索框（Notion 风格）

全局播放器（收起 / 展开）




---

五、全局搜索框（完全对标 Notion）

1. 位置与形态

固定在顶部

宽度：内容区 100%

高度：40px


2. 样式规范

属性	值

背景	#F7F7F5
圆角	6px
边框	无
文字色	#2F3437
Placeholder	#9B9FA4


3. 状态变化

默认：#F7F7F5

Hover：#EFEFEE

Focus：背景不变，仅 caret


> ❗ Focus 不画蓝框、不高亮描边




---

六、列表类页面（搜索 / 收藏 / 历史）统一规范

1. 列表项是「行」，不是「卡片」

无边框

无卡片阴影

靠 hover 区分


2. 列表项样式

属性	值

高度	48–56px
内边距	12px 16px
背景	透明
Hover	#EFEFEE
Active	#E6E6E4


> 列表项之间不画分割线




---

七、播放器设计（重点）

1. 收起态播放器（Mini Player）

固定底部

高度：56px

背景：#FFFFFF

顶部 可选 使用极弱 shadow


0 -1px 2px rgba(0,0,0,0.04)

不画上边框


2. 展开态播放器（Full Player）

覆盖主内容

背景：#FFFFFF

信息分区：靠间距，不靠线


3. 控件原则

图标按钮无边框

Hover 才显示背景灰



---

八、账号 / 登录页

1. 定位

极简

不做品牌化 UI

更像 Notion 的设置页


2. 表单规则

输入框：

背景灰

无边框


错误提示：文字，不画红框



---

九、给开发/设计人员的硬性约束（非常重要）

必须遵守

❌ 不允许“觉得不好看就加条线”

❌ 不允许随意加阴影

❌ 不允许自选灰色


遇到分不清层级时

> 先加 padding，其次改背景，最后才考虑 shadow




---

十、判断是否「像 Notion」的自检清单

关掉 hover 后，页面是否几乎全白？

截图后是否很难看到“线”？

是否主要靠鼠标移动才能看到结构？


如果答案是「是」👉 基本对了。
